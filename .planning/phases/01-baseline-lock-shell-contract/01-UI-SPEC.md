## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, badge padding trim, micro offsets |
| sm | 8px | Compact chips, meta rows, tight list spacing |
| md | 16px | Default card padding, nav item padding, form field rhythm |
| lg | 24px | Section header spacing, standard card interiors |
| xl | 32px | Split-layout gaps, hero/stat clustering, large card padding |
| 2xl | 48px | Major section header to content break, large CTA padding |
| 3xl | 64px | Page-level section separation and desktop shell breathing room |

Exceptions: sticky header height 64px mobile / 80px desktop, control heights 40px and 48px, and hero-top offset 80px. These are baseline-derived and still remain multiples of 4.

Source notes: Derived from baseline utility usage such as `gap-4`, `gap-6`, `p-6`, `py-16`, `py-24`, `lg:py-28`, `lg:py-32`, `h-16`, and `lg:h-20` in the pinned bundle.

## Route And Shell Contract

### Baseline semantic routes

| Baseline route | Scoped reconstruction route | State contract |
|---------------|-----------------------------|----------------|
| `/` | `/entry-station` | Home page with full landing shell |
| `/about` | `/entry-station/about` | Independent About page, not collapsed into Home |
| `/products` | `/entry-station/products` | Independent Products page with tab switch |
| `/franchise` | `/entry-station/franchise` | Independent Franchise page with calculator and form |
| `/news` | `/entry-station/news` | News list page |
| `/news?id={id}` | `/entry-station/news?id={id}` | News detail state with return-to-list affordance |

### Host shell rules

- Public host entry remains `/`.
- `/` continues to render the landing through the existing iframe shell path.
- The landing runtime remains a five-destination experience, not a single long compliance page.
- Academy entry remains a shell-visible behavior, not a baseline content requirement. The landing may trigger academy through shell glue, but later phases must not hard-code shell internals as part of the restored landing design.
- Academy can return to the entry shell using the current host behavior.

### Shared landing shell

- Fixed top navigation across all landing routes.
- Desktop nav layout: brand lockup left, pill-style route cluster center, theme toggle plus contact CTA right.
- Mobile nav layout: brand lockup left, theme toggle and hamburger right, full-screen dim overlay with floating rounded menu card below the header.
- Scroll-reactive nav: transparent over hero, then blurred/lightened shell after roughly 50px of scroll.
- Footer appears across all five landing destinations and includes brand summary, company links, support links, legal links, contact details, and a circular scroll-to-top button.
- Theme toggle persists user choice via `localStorage("theme")` and applies `data-theme` on the root element. Default theme is light.

Source notes: `EntryShell.tsx`, `01-RESEARCH.md`, baseline `src/App.tsx`, `src/sections/Navigation.tsx`, `src/sections/Footer.tsx`, and `src/contexts/ThemeContext.tsx` markers preserved inside the approved JS bundle.

## Visual Direction

- Overall tone is premium health-tech with strong consumer-marketing energy, not sterile clinical compliance design.
- Light mode is the default baseline feel. It uses airy off-white backgrounds, glassy white cards, saturated green accents, soft black shadows, and Apple-like pill/button treatments.
- Dark mode is also part of the approved baseline. It uses deep forest-black surfaces, muted sage accents, translucent glass panels, and luminous radial glow layers.
- Full-bleed photography is essential to the system. Do not replace major image panels with icon blocks, abstract blobs, or text-only compliance cards.
- The primary focal anchor is the Home hero pairing of the `AI细胞修复` / `告别亚健康` headline with its adjacent hero imagery; restoration work must keep that headline-image lockup as the first visual emphasis.
- Rounded geometry is prominent: 16px, 24px, and 32px radii dominate pills, cards, modals, and image frames.
- Blur and translucency are part of the brand feel. Frosted nav shells, glass cards, and soft radial background glows are baseline behavior, not embellishments.
- Large display typography is required on hero and page headers. Do not reduce the experience to small headings and dense body copy.
- Green is a highlight color, not a page wash. The baseline relies on neutral or dark surfaces with green emphasis.

Source notes: Baseline CSS tokens, `apple-card`, `apple-button`, `glass-effect`, `text-gradient`, hero backgrounds, and section-level gradient usage in the approved bundle.

## Interaction And Accessibility Contract

- Preserve visible focus outlines. Baseline uses a 2px green focus ring/outline for buttons, links, and fields.
- Preserve smooth scrolling and the footer scroll-to-top control; the circular control must expose readable text or an `aria-label` fallback.
- Preserve text selection tinting in green.
- Preserve reduced-motion fallback behavior from the baseline CSS for users who prefer reduced motion.
- Preserve card hover lift behavior on desktop. Do not flatten all cards to static panels.
- Preserve image zoom-on-hover behavior for news and case cards.
- Preserve desktop and mobile navigation active-state styling with filled green pill treatment on the active item.
- Preserve theme toggle affordances: sun/moon icon swap and `切换到日间模式` / `切换到夜间模式` labels.

Source notes: Approved baseline CSS and section/component markers inside the approved JS bundle.

## Do Not Redesign

- Do not collapse the five-page experience into one long scroll page.
- Do not replace the premium glass-and-photography look with flat white compliance panels.
- Do not switch the target typography to generic system fonts or shrink the hero into a weak marketing banner.
- Do not remove the theme toggle or dark-mode route parity.
- Do not replace large photo panels with icon-only cards.
- Do not rewrite the information hierarchy to foreground compliance disclaimers or soft-medical copy.
- Do not normalize baseline CTA wording, phone numbers, or contact emails unless a later approved content phase explicitly says to do so.
- Do not introduce shadcn defaults, dashboard chrome, or a new design language in the name of maintainability.

## Fidelity Rubric For Later Phases

- A reconstruction passes only if reviewers can identify the five distinct destination states and their original section ordering without consulting the old runtime.
- A reconstruction fails if it visually resembles the current compliance-safe single-page direction more than the approved pre-`7be7097^` bundle.
- Differences are acceptable only when they are required by `/entry-station` route scoping or by shell-to-academy glue. They are not acceptable when they simplify, modernize, flatten, or sanitize the original landing design.
- Theme parity is required. A light-only rebuild is not baseline-equivalent.
- Home interaction parity is required for the expandable advantages section, the process-step switching/auto-advance, animated proof numbers, and preview-to-detail news navigation.
- News parity is required for the list state, filter/search controls, `?id=` detail state, and return path.
- Footer and navigation must remain shared across all landing destinations.
