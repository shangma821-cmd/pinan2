Pre-computed audit result (grep confirmed during planning):
  - Zero var() references in landing.css point to academy-owned unprefixed variables.
  - Name collision check: no --landing-* name overlaps with --bg-0, --text, --green, etc.

Existing isolation mechanisms already in landing.css:
  - .landing-app { color: var(--landing-text-primary); } — overrides body { color: var(--text) }
  - body:has(.landing-app) { background: var(--landing-bg-primary); } — overrides academy body background
  - .landing-app { font-family: 'Noto Sans SC', sans-serif; } — overrides body { font-family }
  - .landing-shell { isolation: isolate; } — stacking context isolation

Known gap to verify:
  - Landing children could inherit academy body font-family if they explicitly unset or override
    font-family without using a landing-scoped selector. Confirm no such patterns exist in
    landing components.
